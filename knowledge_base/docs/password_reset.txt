RUNBOOK: Password Reset / Account Lockout
Common Causes:
1. Too many failed login attempts (auto-lock after 5 tries)
2. Password expired under 90-day rotation policy
Resolution Steps:
1. Verify employee identity using employee ID
2. Reset password via Active Directory
3. Send temporary password to registered mobile number
4. Require password change on next login
5. If repeated lockouts from unknown IPs, escalate to L2-Security
